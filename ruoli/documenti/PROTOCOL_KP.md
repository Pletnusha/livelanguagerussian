# COPYWRITER PROTOCOL — COMMERCIAL PROPOSAL (КП)

Universal protocol. Applies to any product, any audience, any format.

---

## PRINCIPLES (never violated)

- A commercial proposal is one page. Not a brochure. Not a presentation.
- Language: impersonal constructions only. No "tu/tuo", no "Lei/Sua".
- No positioning against competitors — the document speaks only about what is offered.
- No hype language ("innovativo", "all'avanguardia", "rivoluzionario") — prohibited.
- One CTA. One action. Zero alternatives.
- Placeholders `[IN_BRACKETS]` wherever data is missing. Never invent.

---

---

## STEP 1 — COORDINATE WITH ID

Request sent **before writing** program and output sections.

Required questions:
1. Exact product structure (blocks, stages, modules)
2. Specific audience procedures / use cases the practice is built on
3. What the client / participant takes away — concrete and tangible

---

## STEP 2 — COORDINATE WITH SELLER

Request sent **before writing** positioning and closing sections.

Required questions:
1. Who decides — individual or organization?
2. Most likely objections from this audience
3. Social proof — testimonials, case studies, references

---

## STEP 3 — DOCUMENT STRUCTURE

```
[HEADER]
Name, title, contact | Product name, format, key parameters

[OPENING LINE — 2-3 sentences]
Shows understanding of the audience's professional context.
Does not explain their problem to them — they are professionals.
Moves to the offer in the same paragraph.

[PRODUCT DESCRIPTION]
What it is. How it works. Why this approach.
Maximum 3 short paragraphs.

[PROGRAMMA / STRUTTURA]
Bulleted list of concrete blocks.
No timings unless confirmed by ID.

[OUTPUT / RISULTATI]
What the client receives at the end.
Only what is realistically achievable.

[DETTAGLI PRATICI]
Format | Duration | Participants | Tools | Date | Price

[CTA]
One sentence. One action. One contact.

[NOTE / PRIVACY]
Confidentiality and GDPR — as a trust signal, not a warning.
```

---

## STEP 4 — WRITING RULES

**Opening line:**
- Demonstrates understanding of the audience's professional context
- Does not explain their problem to them — they already know it
- Does not position against competitors
- Moves to the solution in the same paragraph

**Program / structure:**
- Each bullet = a concrete block with clear content
- Timings only if confirmed by ID
- Format: "What — why / what the outcome is"

**Output:**
- Only what is realistically achievable within the time and format
- Skill > artifact when time is limited
- Tangible deliverable (guide, checklist, template) confirmed with ID

**CTA:**
- One action: write / call / fill in a form
- One contact
- No alternatives, no menu of options

**Privacy / GDPR:**
- Always at the end as a professional standard signal
- Framing: what is protected, how, by whom
- Not a disclaimer — part of the positioning

## FORMAT DECISION MATRIX

Before writing, identify which scenario applies. This determines priorities for structure, tone, and visual weight.

---

### SCENARIO A — Logic-first (B2B, organization, committee)

**Typical audience:** legal associations, HR departments, corporate buyers, procurement committees
**Who decides:** rarely one person — document circulates, gets forwarded
**What they look for:** what, how much, why, what the outcome is — fast, scannable
**Risk if wrong:** decorative formatting reads as lack of seriousness

**Priorities:**
- Structure is rigid and predictable — no surprises
- Information hierarchy is explicit: problem → solution → program → output → price → CTA
- Tone: precise, impersonal, zero inflation
- Visual design: clean, minimal, functional — color only to guide the eye
- Length: one page, no exceptions
- Form serves structure. Never competes with it.

---

### SCENARIO B — Form-first (B2C, personal brand, creative / luxury)

**Typical audience:** individuals making personal decisions, creative professionals, high-end clients
**Who decides:** the person reading it — emotional and aesthetic judgment matters
**What they look for:** feeling, identity match, trust through quality of execution
**Risk if wrong:** a generic-looking document signals generic-quality work

**Priorities:**
- Visual design is part of the argument — it demonstrates competence
- Typography, spacing, and color carry meaning
- Tone can be warmer, more direct, even with personality
- Structure is still present but less rigid — can open with a statement, a tension, an image
- Length: one page, but breathing room matters more than density
- Form IS content. A poorly designed КП is already an objection.

---

### SCENARIO C — Hybrid (professional individual, warm contact, BNI context)

**Typical audience:** solo professionals, studio owners, individual practitioners reached through referral
**Who decides:** one person, but with professional judgment — not purely emotional
**What they look for:** trust signal + clarity + personal fit
**Risk if wrong:** too corporate feels cold; too creative feels unserious

**Priorities:**
- Structure follows Scenario A logic (clear, scannable, one page)
- Visual design follows Scenario B standards (not generic, not decorated)
- Tone: professional but human — impersonal constructions, but not bureaucratic
- The referral context should be acknowledged somewhere (even subtly)
- Form must be good enough to not create doubt. It does not need to impress.

---

**Decision rule:** when in doubt, default to Scenario A structure + Scenario C visual standard.
Never default to Scenario B unless the audience and product explicitly call for it.



| Scenario | Range |
|---|---|
| B2B, organization pays for group, warm contact | €1,200–1,500 |
| B2B, cold outreach, first contact | €800–900 |
| B2B, strong referral, professional audience | up to €2,000 |
| B2C offline, BNI members | €100 / participant, max 8 |
| B2C offline, external participants | €150 / participant, max 8 |
| B2C online | €80 / participant, max 20 |
